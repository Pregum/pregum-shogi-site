# CPU対戦AIの実装ナレッジ（2026-07）

対象: `src/client/lib/ai.ts`（約150行）
関連: `src/shared/shogi.ts`（`pseudoMoves` / `legalMoves`）、`src/client/pages/Cpu.tsx`（UI側）、`src/client/lib/ai.test.ts`（テスト）

CPU対戦のAIは**すべてブラウザ内で動く**。サーバー（Cloudflare Workers）には一切負荷がかからず、無料枠の消費もゼロ。

## 1. 基本アルゴリズム: ネガマックス探索 + アルファベータ枝刈り

「自分が指す → 相手が最善で返す → また自分が指す…」を木構造で先読みし、**最終局面の評価値が最も良くなる手**を選ぶ（`negamax()`）。

- **ネガマックス**: 「相手にとっての最善 = 自分にとっての最悪」という二人零和ゲームの性質を使い、手番を入れ替えながら評価値の符号を反転させて再帰するミニマックス法の実装スタイル
- **アルファベータ枝刈り**: 「この枝はもう調べても結論が変わらない」と分かった時点で読みを打ち切る高速化。**駒得が大きい手から先に調べる**ようソートすることで枝刈り効率を上げている（`captureValue` でソート）

## 2. 評価関数（局面の点数付け）

`evaluate()` は局面を次の合計点で採点する:

- **駒の価値**: 歩90・香315・桂405・銀495・金540・角855・飛990点…（プロの将棋AIでよく使われる比率）。成駒は高め（と金560・馬1050・龍1190）
- **持ち駒**も同じ価値でカウント（駒を取ると評価が上がる）
- **前進ボーナス**: 駒が敵陣に近いほど +2点/段。序盤に駒を進める動機付け

つまり本質的には「**駒の損得勘定で先読みするAI**」。囲いや手筋の知識は持っていない。

## 3. 強さ3段階の正体

| レベル | 読みの深さ | ランダムゆらぎ |
| --- | --- | --- |
| こぎつね（弱い） | 1手（自分の手だけ） | ±260点 — 歩損レベルのミスをよくする |
| きつね（ふつう） | 2手（相手の応手まで） | ±40点 — たまに小さなミス |
| 妖狐（強い） | 3手（自分→相手→自分） | なし — 読みの範囲で常に最善 |

「ゆらぎ」は評価値に乱数を加算して、わざと最善以外の手も選ばせる弱体化手法。妖狐は3手読みなので**1手詰みを逃さず、タダ捨てには必ず反撃**する（頭金の詰みを発見できることをユニットテストで担保: `ai.test.ts`）。

## 4. 高速化の工夫

将棋は1局面あたり合法手が100手前後あるため、素朴な3手読みは 100³ = 100万局面になる。対策:

- **探索中は厳密な合法手判定を省略**（`pseudoMoves`）。王手放置などの違法手もいったん読み、「相手に王を取られる展開になったら大負け（±INF）」と評価することで、結果的に違法手が選ばれない仕組み（将棋AIの定石テクニック）。厳密な判定は1手ごとに全候補手を仮に指して王手チェックが必要で重いため、これで数十倍速くなる
- **ルート（実際に指す手）だけは厳密な `legalMoves` から選ぶ**ので、反則手は絶対に指さない
- 保険として **40万ノードで探索を打ち切る**上限つき（`SearchCtx.budget`）。スマホでも1〜2秒以内に着手する
- UIを固めないよう `setTimeout` で1フレーム譲ってから思考開始。UX上「考えています…」の演出として最低500msの間を置いている（`Cpu.tsx` 側）

## 5. 対人戦との実装の違い

| | 対人戦 | CPU戦 |
| --- | --- | --- |
| 手の検証 | サーバー（Durable Object）で `legalMoves` 照合 | クライアント内で完結 |
| 待った | 相手の承認制 | 自由（即時2手戻し） |
| 千日手・詰み判定 | サーバーで判定 | クライアントで判定（同じ `shogi.ts` を共用） |
| 棋譜保存 | 終局時に localStorage | 同じ |

ルールエンジン `src/shared/shogi.ts` をクライアント・サーバーで共用しているため、判定ロジックの二重実装がない。

## 6. さらに強くしたくなったら（未実装の定番手法）

1. **反復深化 + 置換表**: 深さ4〜5手へ。同一局面の再探索をハッシュでスキップ
2. **静止探索（quiescence search）**: 駒の取り合いが終わるまで読みを延長し、水平線効果を軽減
3. **Web Worker化**: 思考をバックグラウンドスレッドへ移し、UIのフリーズリスクをゼロに
4. **定跡データ**: 序盤数手を定跡から指させて自然な立ち上がりに
