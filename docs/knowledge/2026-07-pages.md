# 各ページとルーティングの構成（2026-07）

対象: `src/client/App.tsx`（ルーティング）、`src/client/pages/*.tsx`（各ページ）、`src/client/lib/router.tsx`（自前ルーター）

## ルーティングの仕組み

React Router などのライブラリは使わず、**約35行の自前ルーター**（`router.tsx`）で実現している:

- `navigate()` が `history.pushState` + `popstate` イベント発火、`usePath()` がパスを購読、`<Link>` がクリックを乗っ取る、の3点セット
- `App.tsx` がパスを正規表現でマッチして表示ページを切り替える
- **SPAフォールバック**: 直接 `/room/xxx` を開いてもHTMLが返るのは、wrangler の assets 設定 `not_found_handling: "single-page-application"` のおかげ（`wrangler.jsonc`）

依存を増やさなかった理由: ページ数が少なく、ネスト・ガード・遅延読み込みが不要なため。

## 各ページの要点

### Home（`/`）
- 名前は localStorage（`kitsune-shogi:name`）に保存し全ページで共有
- 「対局部屋を作る」→ `POST /api/rooms` でID発行 → `/room/:id` へ遷移。**選択した持ち時間は sessionStorage 経由で Room に渡す**（部屋作成者のjoinメッセージにだけ載せるため）
- 入室欄は招待URLでもID単体でも受け付ける（正規表現で吸収）

### Room（`/room/:id`）
- 名前未設定なら入室前に名前入力画面を挟む（招待された側の初回動線）
- `useRoom` フックが WebSocket 接続・自動再接続・状態購読を担い、ページは受け取った `GameSnapshot` を描画するだけ
- 対局終了時に棋譜を localStorage へ自動保存（`saveKifu`。roomId+gameNo をキーに重複保存を防止）
- 待った/再戦の承認モーダル、持ち時間のカウントダウン表示（500ms間隔で再描画）、効果音の発火もここ

### Cpu（`/cpu`）
- サーバー不使用。`useState` で棋譜を持ち、AI（`lib/ai.ts`）が相手の手を返す
- 終局判定（詰み・千日手）はクライアントで `outcome`/`checkRepetition` を直接呼ぶ
- 待ったは承認不要で「自分の手番に戻るまで」即時巻き戻し

### Practice（`/practice`）
- 手番に関係なく両陣営を操作できる（`interactive` を常にON、`legalMoves` はそのまま使うので反則手は指せない）
- 「1手戻す」「盤を初期化」のみのシンプル構成。詰みになったら勝敗を表示

### Rules（`/rules`）
- 駒の動き図解は画像ではなく**5×5のCSSグリッドをデータ駆動で描画**（`DIAGRAMS` 配列に `[row, col, マーク]` を列挙）。テーマカラーに追従し、追加・修正が容易
- サイトの操作方法（タップ操作・待った・棋譜保存）もここに集約

### Kifu（`/kifu`, `/kifu/:id`）
- 一覧: localStorage の棋譜を日付降順で表示。KIFダウンロード・削除（2段階確認）
- ビューア: `replayAll` で全局面を先に計算し、スライダー的に手数を移動。**矢印キー(←/→)対応**。棋譜リストのクリックでその手へジャンプ

## ページ共通の部品

| ファイル | 役割 |
| --- | --- |
| `components/Board.tsx` | 盤面。駒の選択→移動先ハイライト→着手、成りダイアログ、持ち駒、視点反転（後手は盤が逆さになる）を1コンポーネントで提供。対人戦/CPU/練習/再生の全ページで共用 |
| `components/FoxMark.tsx` | 狐ロゴ・鳥居のSVG |
| `lib/storage.ts` | localStorage の入出力（棋譜・名前・トークン）と KIF ダウンロード |
| `lib/sound.ts` | 効果音（Web Audio合成）。ON/OFF はヘッダーの🔊トグル |
| `styles.css` | 全スタイル。「稲荷の黄昏」テーマのCSS変数（`--vermilion` 等）を定義 |

## 設計上のポイント

- **盤面ロジックはページに持たせない**: どのページも「`Position` を渡して `Move` を受け取る」だけ。ゲームの正しさは `shared/shogi.ts` が一元管理
- **対局モードの違いは「手の適用先」の違いだけ**: 対人戦=WebSocketで送る / CPU戦=ローカルstateに積んでAIを呼ぶ / 練習=ローカルstateに積むだけ。Boardから見れば全部同じ
