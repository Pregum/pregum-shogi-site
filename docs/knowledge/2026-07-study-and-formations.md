# 勉強画面と囲い完成エフェクトの実装（2026-07）

対象: `src/shared/formations.ts`（囲い検出）、`src/client/lib/lessons.ts`（レッスンデータ）、`src/client/components/LessonBoard.tsx` / `FormationBanner.tsx`、`src/client/pages/Study.tsx`

## 勉強画面（/study）— GIFではなく「動く盤面」

手筋・囲いの解説はGIF画像ではなく、**既存のBoardコンポーネントをコマ送り再生する**方式を採用した。

- 利点: 画像アセット不要で軽量 / テーマ配色に自動追従 / レッスンの追加・修正がデータ1件で済む / 将来ステップに注釈やインタラクションを足せる
- `Lesson` = タイトル + 概要 + `LessonStep[]`（局面・キャプション・直前の着手マス）。`LessonBoard` が2.4秒間隔で自動再生し、手動の戻る/進む/一時停止も可能

### レッスンデータの作り方（2方式）

| 方式 | 用途 | 仕組み |
| --- | --- | --- |
| `seq()` | 手筋（ふんどしの桂・割り打ちの銀・田楽刺し・頭金） | 実際の `applyMove` で1手ずつ進める。**テストで全手順が `legalMoves` に含まれることを検証**しており、将棋として正しいことが保証される |
| `frames()` | 囲い（矢倉・美濃・穴熊） | 駒を1枚ずつ動かすキーフレーム。手番の概念を無視できるので「自分の駒だけ動かして相手の手は省略」という解説向けの見せ方ができる |

## 囲い完成エフェクト

- `src/shared/formations.ts` に囲いを「先手視点の絶対座標 + 駒種」のパターンとして定義（矢倉=玉88金78銀77金67 / 美濃=玉28銀38金58金49 / 穴熊=玉99香98銀88金79）。後手は盤を点対称に反転して同じパターンで照合
- `useFormationBanner` フック（クライアント）が局面を監視し、**新しく**完成した囲いだけバナー+ファンファーレで祝う
  - 対局ごとの `Set` で発火済みを記録（組み直しでの再発火を防止）
  - **初回観測時は黙って記録だけ**する — リロードで途中参加したときに、すでにある囲いを祝わないため
  - 再戦（gameNo変化）・CPU戦の「もう一局」でリセット
- 対人戦(Room)とCPU戦(Cpu)の両方に同じフックを組み込み。検出は共有層なので、将来サーバー側で統計を取ることも可能

## 設計メモ

- 囲い判定は歩の位置を要求しない緩め設計（歩が突いてあっても完成扱い）。誤検出よりも「せっかく組んだのに出ない」ほうが体験が悪いため
- 勉強画面の囲いレッスン最終形が検出パターンと一致することをテストで担保 → レッスンとエフェクトの内容がズレない
- 今後の拡張候補: 戦法の検出（棒銀・四間飛車など、飛車の筋や銀の進出で判定）、囲いの追加（銀冠・エルモ囲いなど）、手筋レッスンの追加（十字飛車・継ぎ歩など）
